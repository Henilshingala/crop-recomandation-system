### 3.6 — Asymmetric Noise Rules (Improvement #4)

**v2.1 approach (REPLACED):** Uniform symmetric 5% noise in all directions for all clusters.

**v2.2 approach — directional per-cluster noise rules:**

ASYMMETRIC_NOISE_RULES = {
    "A_tropical_wet": {

    },

    "temp_shift": (-2, 2),
    "rain_shift": (-0.10, 0.10),
    "hum_shift":  (-4, 4),

**Noise injection function:**

```python
def inject_asymmetric_noise(df: pd.DataFrame, fraction: float = 0.05) -> pd.DataFrame:
    df = df.copy()
    n_noise = int(len(df) * fraction)
    noise_idx = np.random.choice(len(df), size=n_noise, replace=False)

    for idx in noise_idx:
        crop = df.loc[idx, "label"]
        cluster = CROP_TO_CLUSTER.get(crop, "")
        rules = ASYMMETRIC_NOISE_RULES.get(cluster, DEFAULT_NOISE_RULE)

        # Temperature: asymmetric shift
        t_lo, t_hi = rules["temp_shift"]
        df.loc[idx, "temperature"] += np.random.uniform(t_lo, t_hi)

        # Humidity: asymmetric shift
        h_lo, h_hi = rules["hum_shift"]
        df.loc[idx, "humidity"] += np.random.uniform(h_lo, h_hi)

        # Rainfall: proportional asymmetric shift
        r_lo, r_hi = rules["rain_shift"]
        df.loc[idx, "rainfall"] *= (1 + np.random.uniform(r_lo, r_hi))

        # pH: small symmetric noise
        df.loc[idx, "ph"] += np.random.uniform(-0.3, 0.3)

    # Re-clip to physical limits
    df["temperature"] = df["temperature"].clip(0, 50)
    df["humidity"]    = df["humidity"].clip(0, 100)
    df["rainfall"]    = df["rainfall"].clip(10, 3500)
    df["ph"]          = df["ph"].clip(3.5, 9.5)

    return df
```

### 3.7 — SIGMA_SCALE (Gaussian Tightening)

**Purpose:** Reduce within-class Gaussian noise to create cleaner class boundaries, boosting accuracy while leaving room for soil/irrigation features.

```python
SIGMA_SCALE = 0.87
```

Every `sample_gaussian()` call uses `std * SIGMA_SCALE` instead of raw `std`.

### 3.8 — Irrigation Probability Polarization

**Purpose:** Push moderate irrigation probabilities toward 0 or 1, making irrigation more predictive of crop identity.

```python
IRRIG_POLARIZE_STRENGTH = 3.5

def polarize_irrig_prob(p: float, strength: float = IRRIG_POLARIZE_STRENGTH) -> float:
    """Push irrigation probability toward 0 or 1 using power transform."""
    if p <= 0 or p >= 1:
        return p
    if p < 0.5:
        return (p / 0.5) ** strength * 0.5
    else:
        return 1.0 - ((1.0 - p) / 0.5) ** strength * 0.5
```

**Effect with strength=3.5:**

| Original Prob | Polarized Prob |
|--------------|----------------|
| 0.05 | 0.001 |
| 0.10 | 0.004 |
| 0.20 | 0.025 |
| 0.30 | 0.074 |
| 0.40 | 0.152 |
| 0.50 | 0.500 |
| 0.60 | 0.848 |
| 0.70 | 0.926 |
| 0.80 | 0.975 |
| 0.90 | 0.996 |
| 0.95 | 0.999 |

### 3.9 — Irrigation Subpopulation Generation (5-Feature Split)

**Critical design decision:** Instead of applying post-hoc additive shifts to features, irrigation status determines the **Gaussian mean** during generation. This creates genuinely separate distributions that the model must use irrigation to explain.

**Constants:**

```python
# High-importance features
IRRIG_HUM_SHIFT    = 25.0    # irrigated → higher humidity
IRRIG_N_SHIFT      = 20.0    # irrigated → higher N
IRRIG_RAIN_SCALE   = 0.82    # irrigated → less rain needed
RAINFED_HUM_SHIFT  = -10.0   # rainfed → lower humidity
RAINFED_N_SHIFT    = -7.0    # rainfed → lower N
RAINFED_RAIN_SCALE = 1.08    # rainfed → more rain dependent

# Lower-importance features (forces model to use irrigation directly)
IRRIG_TEMP_SHIFT   = -5.0    # irrigated → evaporative cooling
RAINFED_TEMP_SHIFT = +2.0    # rainfed → warmer micro-climate
IRRIG_PH_SHIFT     = +0.8    # irrigated → water buffering raises pH
RAINFED_PH_SHIFT   = -0.3    # rainfed → slightly more acidic
```

### 4.4 — Non-Uniform Geographic Test (Improvement #5)

### Key Discoveries During Tuning

1. **SIGMA_SCALE was the single biggest accuracy lever** — Run 6 jumped from 86.95% to 90.40% just by tightening Gaussian distributions
2. **Subpopulation approach was the key for irrigation** — Generating separate Gaussians for irrigated/rainfed gave higher importance than post-hoc additive shifts
3. **Polarization was the final piece for irrigation** — Power-transforming irrigation probabilities toward 0/1 made the binary feature more crop-specific
4. **Soil and irrigation importance compete** — Both draw from the same "importance budget" in Random Forest; larger soil shifts slightly reduce irrigation importance and vice versa
5. **Binary features have an inherent ceiling in RF MDI importance** — The theoretical maximum for a binary feature in a 10-feature RF is much lower than continuous features
